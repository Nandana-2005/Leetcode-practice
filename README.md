# LeetCode Practice

A structured repository dedicated to documenting Data Structures and Algorithms (DSA) solutions using Java.

---

## Repository Overview

This repository serves as a personal log of problem-solving progress on LeetCode. The focus is on maintaining clean code architecture, optimizing time and space complexity, and tracking algorithmic patterns.

* **Language:** Java (`.java`)
* **Focus:** Code readability, optimal complexity, and conceptual clarity.
* **Documentation:** Every solution includes precise inline comments explaining the core logic and approach wherever necessary.

---

## Repository Structure

Solutions are organized strictly by their respective data structures and algorithmic paradigms for seamless navigation:

| Category | Description |
| :--- | :--- |
| `Backtracking` | Combinatorial search, permutations, and subsets. |
| `Binary-Search` | Logarithmic search space reduction techniques. |
| `Dynamic-Programming` | Optimization problems using memoization and tabulation. |
| `Graph` | Non-linear data structures, DFS, BFS, and traversals. |
| `LinkedLists` | Pointer-based manipulations, reversals, and cycle detections. |
| `Sorting` | Fundamental sorting algorithms and optimizations. |
| `Stack` | Linear data structures dealing with LIFO operations. |
| `Strings` | String manipulation, pattern matching, and parsing. |
| `Trees` | Hierarchical structures, binary trees, and traversals. |
| `hashMap` | Frequency counting, hashing techniques, and fast lookups. |

---

## Usage

1. **Browse by Topic:** Navigate to any specific directory above to explore problems related to that domain.
2. **Review the Code:** Open any `.java` file to view the implementation alongside inline comments detailing the thought process.
